   73: 
   74: C<Test::Mockingbird::TimeTravel> provides a lightweight, deterministic
   75: time-control layer for Perl tests. It allows you to freeze time, move it
   76: forward or backward, jump to specific timestamps, and run code under a
   77: temporary time override - all without touching Perl's built-in C<time()>
   78: or relying on global monkey-patching.
   79: 
   80: The module is designed for test suites that need:
   81: 
   82: =over 4
   83: 
   84: =item *
   85: 
   86: predictable timestamps
   87: 
   88: =item *
   89: 
   90: repeatable behaviour across runs
   91: 
   92: =item *
   93: 
   94: clean separation between real time and simulated time
   95: 
   96: =item *

   99:

  101: 
  102: Unlike traditional mocking of C<time()>, TimeTravel does not replace Perl's core functions.
  103: Instead, it provides a dedicated C<now()> function
  104: and a small set of declarative operations that manipulate an internal,
  105: frozen clock. This avoids global side effects and makes time behaviour

  734: =head2 restore_all
  735: 
  736: Restore real time and clear all TimeTravel state.
  737: 
  738: =head3 Purpose
  739: 
  740: C<restore_all()> deactivates the TimeTravel engine and returns the system
  741: to normal, non-frozen time.
  742: After calling this function, C<now()> once
  743: again returns the real system time from C<CORE::time()>.
  744: 
  745: This is the canonical way to end a time-travel scenario in tests.
  746: 
  747: =head3 Arguments
  748:

  750: 
  751: =head3 Returns

  758:

  760:

  762: 
  763: =item * Clears C<$BASE_EPOCH>.

  829: =head2 with_frozen_time
  830: 
  831: Temporarily override the TimeTravel clock inside a code block.
  832: 
  833: =head3 Purpose
  834: 
  835: C<with_frozen_time()> runs a block of code under a temporary frozen
  836: timestamp, restoring the previous time state afterward. This allows tests
  837: to simulate nested or scoped time overrides without permanently altering
  838: the global TimeTravel state.
  839: 
  840: It is the safest way to test code that depends on time within a limited
  841: scope, especially when combined with C<freeze_time()>, C<travel_to()>,
  842: C<advance_time()>, or C<rewind_time()>.
  843: 
  844: =head3 Arguments
  845: 
  846:     with_frozen_time($timestamp, $code)
  847: 
  848: Takes two required arguments:
  849: 
  850: =over 4
  851: 
  852: =item * C<$timestamp> - a timestamp in any format supported by
  853: C<_parse_timestamp>, including:
  854: 
  855:     YYYY-MM-DD
  856:     YYYY-MM-DD HH:MM:SS
  857:     YYYY-MM-DDTHH:MM:SSZ
  858:     raw epoch seconds
  859: 
  860: =item * C<$code> - a coderef to execute while the override is active
  861: 
  862: =back
  863: 
  864: =head3 Returns
  865: 
  866: Returns whatever the code block returns. In list context, returns a list.
  867: In scalar context, returns the block's scalar result. In void context,
  868: returns nothing.
  869: 
  870: =head3 Side Effects
  871: 
  872: =over 4
  873: 
  874: =item * Saves the current TimeTravel state (active flag, current epoch,
  875: base epoch).
  876: 
  877: =item * Activates frozen time using the provided timestamp.
  878: 
  879: =item * Executes the code block under the overridden time.
  880: 
  881: =item * Restores the previous TimeTravel state after the block completes,
  882: even if the block throws an exception.
  883: 
  884: =item * Rethrows any exception from inside the block.
  885: 
  886: =back
  887: 
  888: =head3 Notes
  889: 
  890: =over 4
  891: 
  892: =item * C<with_frozen_time()> is fully nestable; each invocation restores
  893: its own state.
  894: 
  895: =item * It does not require time to be frozen beforehand.
  896: 
  897: =item * It is ideal for testing code that performs multiple time-based
  898: operations in different scopes.
  899: 
  900: =back
  901: 
  902: =head3 Example
  903: 
  904:     use Test::Mockingbird::TimeTravel qw(
  905:         now freeze_time with_frozen_time restore_all
  906:     );
  907: 
  908:     freeze_time('2025-01-01T00:00:00Z');

  910: 
  911:     my $inner;
  912:     with_frozen_time '2025-01-02T00:00:00Z' => sub {

  916:     is $inner, 1735776000, 'inner block saw overridden time';
