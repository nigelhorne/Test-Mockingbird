   63: =head1 DESCRIPTION
   64: 
   65: C<Test::Mockingbird::DeepMock> provides a declarative, data-driven way to
   66: describe mocking, spying, injection, and expectations in Perl tests.
   67: 
   68: Instead of scattering C<mock>, C<spy>, and C<restore_all> calls throughout
   69: your test code, DeepMock lets you define a complete mocking plan in a single
   70: hashref, then executes your test code under that plan.
   71: 
   72: This produces tests that are:
   73: 
   74: =over 4
   75: 
   76: =item * easier to read
   77: 
   78: =item * easier to maintain
   79: 
   80: =item * easier to extend
   81: 
   82: =item * easier to reason about
   83: 
   84: =back
   85: 
   86: DeepMock is built on top of L<Test::Mockingbird>, adding structure,
   87: expectations, and a clean DSL.
   88: 
   89: =head1 WHY DEEP MOCK?
   90: 
   91: Traditional mocking in Perl tends to be:
   92: 
   93: =over 4
   94: 
   95: =item * imperative
   96: 
   97: =item * scattered across the test body
   98: 
   99: =item * difficult to audit
  100: 
  101: =item * easy to forget to restore
  102: 
  103: =back
  104: 
  105: DeepMock solves these problems by letting you declare everything up front:
  106: 
  107:     deep_mock(
  108:         {
  109:             mocks        => [...],
  110:             expectations => [...],
  111:         },
  112:         sub { ... }
  113:     );
  114: 
  115: This gives you:
  116: 
  117: =over 4
  118: 
  119: =item * a single place to see all mocks and spies
  120: 
  121: =item * automatic restore of all mocks
  122: 
  123: =item * structured expectations
  124: 
  125: =item * reusable patterns
  126: 
  127: =item * a clean separation between setup and test logic
  128: 
  129: =back
  130: 
  131: =head1 PLAN STRUCTURE
  132: 
  133: A DeepMock plan is a hashref with the following keys:
  134: 
  135: =head2 C<mocks>
  136: 
  137: An arrayref of mock specifications. Each entry is a hashref:
  138: 
  139:     {
  140:         target => 'Package::method',   # required
  141:         type   => 'mock' | 'spy' | 'inject',
  142:         with   => sub { ... },         # for mock/inject
  143:         tag    => 'identifier',        # for spies or scoped mocks
  144:         scoped => 1,                   # optional
  145:     }
  146: 
  147: =head3 Types
  148: 
  149: =over 4
  150: 
  151: =item C<mock>
  152: 
  153: Replaces the target method with the provided coderef.
  154: 
  155: =item C<spy>
  156: 
  157: Wraps the method and records all calls. Must have a C<tag>.
  158: 
  159: =item C<inject>
  160: 
  161: Injects a value or behavior into the target (delegates to C<Test::Mockingbird::inject>).
  162: 
  163: =back
  164: 
  165: =head2 C<expectations>
  166: 
  167: An arrayref of expectation specifications. Each entry is a hashref:
  168: 
  169:     {
  170:         tag   => 'double_spy',   # required
  171:         calls => 2,              # optional
  172:         args_like => [           # optional
  173:             [ qr/foo/, qr/bar/ ],
  174:         ],
  175:     }
  176: 
  177: =head3 Expectation fields
  178: 
  179: =over 4
  180: 
  181: =item C<tag>
  182: 
  183: Identifies which spy this expectation applies to.
  184: 
  185: =item C<calls>
  186: 
  187: Expected number of calls.
  188: 
  189: =item C<args_eq>
  190: 
  191: Arrayref of arrayrefs. Each inner array lists exact argument values expected
  192: for a specific call. Values are compared with C<Test::More::is>.
  193: 
  194: =item C<args_deeply>
  195: 
  196: Arrayref of arrayrefs. Each inner array lists deep structures to compare
  197: against the arguments for a specific call. Uses C<Test::Deep::cmp_deeply>.
  198: 
  199: =item C<args_like>
  200: 
  201: Arrayref of arrayrefs of regexes. Each inner array describes expected
  202: arguments for a specific call.
  203: 
  204: =item C<never>
  205: 
  206: Asserts that the spy was never called.
  207: Mutually exclusive with C<calls>.
  208: 
  209: =item C<order>
  210: 
  211: Asserts that the listed fully-qualified method names were called in the
  212: given left-to-right order (not necessarily consecutively). This key is
  213: processed once per expectations list, not per spy tag. It delegates to
  214: C<Test::Mockingbird::assert_call_order>.
  215: 
  216:     expectations => [
  217:         { order => ['A::fetch', 'B::process', 'C::save'] },
  218:     ]
  219: 
  220: =back
  221: 
  222: =head2 C<globals>
  223: 
  224: Optional hashref controlling global behavior:
  225: 
  226:     globals => {
  227:         restore_on_scope_exit => 1,   # default
  228:     }
  229: 
  230: =head2 C<time>
  231: 
  232: Optional hashref describing a time-travel plan to apply while the
  233: C<deep_mock> block is running. This integrates with
  234: L<Test::Mockingbird::TimeTravel> and allows declarative control of frozen
  235: time, time jumps, and temporal overrides.
  236: 
  237: If provided, the time plan is applied:
  238: 
  239: =over 4
  240: 
  241: =item 1. before any mocks are installed
  242: 
  243: =item 2. before the test code block is executed
  244: 
  245: =item 3. automatically restored after the block completes
  246: 
  247: =back
  248: 
  249: A time plan may include any of the following keys:
  250: 
  251:     time => {
  252:         freeze  => '2025-01-01T00:00:00Z',
  253:         travel  => '2025-01-02T12:00:00Z',
  254:         advance => [ 2 => 'minutes' ],
  255:         rewind  => [ 1 => 'hour'    ],
  256:     }
  257: 
  258: =head3 C<freeze>
  259: 
  260: Freezes time at the given timestamp. Accepts any format supported by
  261: C<Test::Mockingbird::TimeTravel>, including:
  262: 
  263: =over 4
  264: 
  265: =item * C<YYYY-MM-DD>
  266: 
  267: =item * C<YYYY-MM-DD HH:MM:SS>
  268: 
  269: =item * C<YYYY-MM-DDTHH:MM:SSZ>
  270: 
  271: =item * raw epoch seconds
  272: 
  273: =back
  274: 
  275: =head3 C<travel>
  276: 
  277: Moves the frozen clock to a new timestamp without unfreezing time.
  278: 
  279: =head3 C<advance>
  280: 
  281: Advances the frozen clock by a duration. Must be an arrayref:
  282: 
  283:     advance => [ $amount => $unit ]
  284: 
  285: Units may be C<seconds>, C<minutes>, C<hours>, or C<days>.
  286: 
  287: =head3 C<rewind>
  288: 
  289: Rewinds the frozen clock by a duration. Same format as C<advance>.
  290:

  495: =head1 TROUBLESHOOTING
  496: 
  497: =head2 "Not enough arguments for deep_mock"
  498: 
  499: You are using the BLOCK prototype form:
  500: 
  501:     deep_mock {
  502:         ...
  503:     }, sub { ... };
  504: 
  505: This only works if C<deep_mock> has a C<(&$)> prototype AND the first
  506: argument is a real block, not a hashref.
  507: 
  508: DeepMock uses C<($$)> to avoid Perl's block-vs-hashref ambiguity.
  509: 
  510: Use parentheses instead:
  511: 
  512:     deep_mock(
  513:         { ... },
  514:         sub { ... }
  515:     );
  516: 
  517: =head2 "Type of arg 1 must be block or sub {}"
  518: 
  519: You are still using the BLOCK prototype form. Switch to parentheses.
  520: 
  521: =head2 "Use of uninitialized value in multiplication"
  522: 
  523: Your spied method is being called with no arguments during spy installation.
  524: Make your method robust:
  525: 
  526:     sub double { ($_[1] // 0) * 2 }
  527: 
  528: =head2 My mocks aren't restored
  529: 
  530: Ensure you didn't disable automatic restore:
  531: 
  532:     globals => { restore_on_scope_exit => 0 }

  534: =head2 Nested deep_mock scopes are not supported
  535: 
  536: DeepMock installs mocks using L<Test::Mockingbird>, which provides only
  537: global restore semantics via C<restore_all>. Because Test::Mockingbird
  538: does not expose a per-method restore API, DeepMock cannot safely restore
  539: only the mocks installed in an inner scope.
  540: 
  541: As a result, nested calls like:
  542: 
  543:     deep_mock { ... } sub {
  544:         deep_mock { ... } sub {
  545:             ...
  546:         };
  547:     };
  548: 
  549: will cause the inner restore to remove the outer mocks as well.
  550: 
  551: DeepMock therefore does not support nested mocking scopes.
