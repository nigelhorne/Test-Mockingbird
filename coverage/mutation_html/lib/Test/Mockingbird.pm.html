  135: =head1 LIMITATIONS
  136: 
  137: =over 4
  138: 
  139: =item C<< ->can() >> may return truthy after unmocking a never-existed method
  140: 
  141: Perl's typeglob (GV) system auto-vivifies a GV entry the first time
  142: C<\&{$full_method}> is called internally (in C<mock()>, C<spy()>, or
  143: C<inject()>). After unmocking, this GV entry remains in the stash with an
  144: "undefined sub" placeholder in the CODE slot. C<< Package->can('method') >>
  145: tests the GV's existence in the stash, not whether the CODE slot is defined,
  146: so it may still return a truthy value.
  147: 
  148: To test whether a sub is callable, use C<defined(&Package::method)> rather
  149: than C<< Package->can('method') >>. C<defined(&...)> correctly returns false
  150: for the placeholder stub. Calling the stub dies with C<"Undefined subroutine">.
  151: 
  152: Deleting the GV from the stash (via C<delete $stash{method}>) would make
  153: C<< ->can() >> return false but would break subsequent mock/inject stacking:
  154: compiled direct calls (C<Package::method()>) cache the GV at compile time,
  155: so a new GV installed after a delete is invisible to those compiled calls.
  156: 
  157: =item Prototype mismatch warning from C<spy()>
  158: 
  159: C<spy()> installs its wrapper directly without going through C<mock()>,
  160: so C<Scalar::Util::set_prototype> is not applied. Wrapping a prototyped
  161: function with C<spy()> still emits a C<Prototype mismatch> warning. Use
  162: C<mock()> with a delegating wrapper if warning-free wrapping is required.
  163: 
  164: =item No nested deep_mock scopes
  165: 
  166: L<Test::Mockingbird::DeepMock> calls C<restore_all()> at scope exit, which
  167: removes every active mock. Nested C<deep_mock> blocks cause the inner exit
  168: to also tear down the outer mocks. Do not nest C<deep_mock> calls.
  169: 
  170: =item Thread safety
  171: 
  172: The internal state (C<%mocked>, C<%mock_meta>, C<@call_log>) is per-process
  173: lexical state. Concurrent threads that install and restore mocks will race.
  174: Do not use this module in threaded test harnesses without external locking.
  175: 
  176: =item Spy return value is a flat list
  177: 
  178: C<spy()> and C<async_spy()> return a coderef that yields a flat list of
  179: call records. A future version may return an arrayref to reduce stack
  180: pressure; the API is not yet changed to avoid breaking callers.
  181: 
  182: =item Private-function encapsulation
  183: 
  184: Functions prefixed with C<_> are private by convention but are not enforced
  185: at runtime (C<Sub::Private> is not activated). White-box tests in C<t/unit.t>
  186: call private functions directly. If C<Sub::Private> enforcement is added, a
  187: testing-interface export mechanism will be required.
  188: 
  189: =back
