  133: =head1 LIMITATIONS
  134: 
  135: =over 4
  136: 
  137: =item C<< ->can() >> may return truthy after unmocking a never-existed method
  138: 
  139: Perl's typeglob (GV) system auto-vivifies a GV entry the first time
  140: C<\&{$full_method}> is called internally (in C<mock()>, C<spy()>, or
  141: C<inject()>). After unmocking, this GV entry remains in the stash with an
  142: "undefined sub" placeholder in the CODE slot. C<< Package->can('method') >>
  143: tests the GV's existence in the stash, not whether the CODE slot is defined,
  144: so it may still return a truthy value.
  145: 
  146: To test whether a sub is callable, use C<defined(&Package::method)> rather
  147: than C<< Package->can('method') >>. C<defined(&...)> correctly returns false
  148: for the placeholder stub. Calling the stub dies with C<"Undefined subroutine">.
  149: 
  150: Deleting the GV from the stash (via C<delete $stash{method}>) would make
  151: C<< ->can() >> return false but would break subsequent mock/inject stacking:
  152: compiled direct calls (C<Package::method()>) cache the GV at compile time,
  153: so a new GV installed after a delete is invisible to those compiled calls.
  154: 
  155: =item Prototype mismatch warning from C<spy()>
  156: 
  157: C<spy()> installs its wrapper directly without going through C<mock()>,
  158: so C<Scalar::Util::set_prototype> is not applied. Wrapping a prototyped
  159: function with C<spy()> still emits a C<Prototype mismatch> warning. Use
  160: C<mock()> with a delegating wrapper if warning-free wrapping is required.
  161: 
  162: =item No nested deep_mock scopes
  163: 
  164: L<Test::Mockingbird::DeepMock> calls C<restore_all()> at scope exit, which
  165: removes every active mock. Nested C<deep_mock> blocks cause the inner exit
  166: to also tear down the outer mocks. Do not nest C<deep_mock> calls.
  167: 
  168: =item Thread safety
  169: 
  170: The internal state (C<%mocked>, C<%mock_meta>, C<@call_log>) is per-process
  171: lexical state. Concurrent threads that install and restore mocks will race.
  172: Do not use this module in threaded test harnesses without external locking.
  173: 
  174: =item Spy return value is a flat list
  175: 
  176: C<spy()> and C<async_spy()> return a coderef that yields a flat list of
  177: call records. A future version may return an arrayref to reduce stack
  178: pressure; the API is not yet changed to avoid breaking callers.
  179: 
  180: =item Private-function encapsulation
  181: 
  182: Functions prefixed with C<_> are private by convention but are not enforced
  183: at runtime (C<Sub::Private> is not activated). White-box tests in C<t/unit.t>
  184: call private functions directly. If C<Sub::Private> enforcement is added, a
  185: testing-interface export mechanism will be required.
  186: 
  187: =back
