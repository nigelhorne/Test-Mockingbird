  139: =head1 LIMITATIONS
  140: 
  141: =over 4
  142: 
  143: =item C<< ->can() >> may return truthy after unmocking a never-existed method
  144: 
  145: Perl's typeglob (GV) system auto-vivifies a GV entry the first time
  146: C<\&{$full_method}> is called internally (in C<mock()>, C<spy()>, or
  147: C<inject()>). After unmocking, this GV entry remains in the stash with an
  148: "undefined sub" placeholder in the CODE slot. C<< Package->can('method') >>
  149: tests the GV's existence in the stash, not whether the CODE slot is defined,
  150: so it may still return a truthy value.
  151: 
  152: To test whether a sub is callable, use C<defined(&Package::method)> rather
  153: than C<< Package->can('method') >>. C<defined(&...)> correctly returns false
  154: for the placeholder stub. Calling the stub dies with C<"Undefined subroutine">.
  155: 
  156: Deleting the GV from the stash (via C<delete $stash{method}>) would make
  157: C<< ->can() >> return false but would break subsequent mock/inject stacking:
  158: compiled direct calls (C<Package::method()>) cache the GV at compile time,
  159: so a new GV installed after a delete is invisible to those compiled calls.
  160: 
  161: =item Prototype mismatch warning from C<spy()>
  162: 
  163: C<spy()> installs its wrapper directly without going through C<mock()>,
  164: so C<Scalar::Util::set_prototype> is not applied. Wrapping a prototyped
  165: function with C<spy()> still emits a C<Prototype mismatch> warning. Use
  166: C<mock()> with a delegating wrapper if warning-free wrapping is required.
  167: 
  168: =item No nested deep_mock scopes
  169: 
  170: L<Test::Mockingbird::DeepMock> calls C<restore_all()> at scope exit, which
  171: removes every active mock. Nested C<deep_mock> blocks cause the inner exit
  172: to also tear down the outer mocks. Do not nest C<deep_mock> calls.
  173: 
  174: =item Thread safety
  175: 
  176: The internal state (C<%mocked>, C<%mock_meta>, C<@call_log>) is per-process
  177: lexical state. Concurrent threads that install and restore mocks will race.
  178: Do not use this module in threaded test harnesses without external locking.
  179: 
  180: =item Spy return value is a flat list
  181: 
  182: C<spy()> and C<async_spy()> return a coderef that yields a flat list of
  183: call records. A future version may return an arrayref to reduce stack
  184: pressure; the API is not yet changed to avoid breaking callers.
  185: 
  186: =item Private-function encapsulation
  187: 
  188: Functions prefixed with C<_> are private by convention but are not enforced
  189: at runtime (C<Sub::Private> is not activated). White-box tests in C<t/unit.t>
  190: call private functions directly. If C<Sub::Private> enforcement is added, a
  191: testing-interface export mechanism will be required.
