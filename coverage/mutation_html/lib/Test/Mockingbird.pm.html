  141: =head1 LIMITATIONS
  142: 
  143: =over 4
  144: 
  145: =item C<< ->can() >> may return truthy after unmocking a never-existed method
  146: 
  147: Perl's typeglob (GV) system auto-vivifies a GV entry the first time
  148: C<\&{$full_method}> is called internally (in C<mock()>, C<spy()>, or
  149: C<inject()>). After unmocking, this GV entry remains in the stash with an
  150: "undefined sub" placeholder in the CODE slot. C<< Package->can('method') >>
  151: tests the GV's existence in the stash, not whether the CODE slot is defined,
  152: so it may still return a truthy value.
  153: 
  154: To test whether a sub is callable, use C<defined(&Package::method)> rather
  155: than C<< Package->can('method') >>. C<defined(&...)> correctly returns false
  156: for the placeholder stub. Calling the stub dies with C<"Undefined subroutine">.
  157: 
  158: Deleting the GV from the stash (via C<delete $stash{method}>) would make
  159: C<< ->can() >> return false but would break subsequent mock/inject stacking:
  160: compiled direct calls (C<Package::method()>) cache the GV at compile time,
  161: so a new GV installed after a delete is invisible to those compiled calls.
  162: 
  163: =item Prototype mismatch warning from C<spy()>
  164: 
  165: C<spy()> installs its wrapper directly without going through C<mock()>,
  166: so C<Scalar::Util::set_prototype> is not applied. Wrapping a prototyped
  167: function with C<spy()> still emits a C<Prototype mismatch> warning. Use
  168: C<mock()> with a delegating wrapper if warning-free wrapping is required.
  169: 
  170: =item No nested deep_mock scopes
  171: 
  172: L<Test::Mockingbird::DeepMock> calls C<restore_all()> at scope exit, which
  173: removes every active mock. Nested C<deep_mock> blocks cause the inner exit
  174: to also tear down the outer mocks. Do not nest C<deep_mock> calls.
  175: 
  176: =item Thread safety
  177: 
  178: The internal state (C<%mocked>, C<%mock_meta>, C<@call_log>) is per-process
  179: lexical state. Concurrent threads that install and restore mocks will race.
  180: Do not use this module in threaded test harnesses without external locking.
  181: 
  182: =item Spy return value is a flat list
  183: 
  184: C<spy()> and C<async_spy()> return a coderef that yields a flat list of
  185: call records. A future version may return an arrayref to reduce stack
  186: pressure; the API is not yet changed to avoid breaking callers.
  187: 
  188: =item Private-function encapsulation
  189: 
  190: Functions prefixed with C<_> are private by convention but are not enforced
  191: at runtime (C<Sub::Private> is not activated). White-box tests in C<t/unit.t>
  192: call private functions directly. If C<Sub::Private> enforcement is added, a
  193: testing-interface export mechanism will be required.
